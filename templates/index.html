Topologie distribuită

Rețeaua este alcătuită din 4 noduri: un sediu central (S1) și 3 filiale regionale (S2 / S3 / S4). Datele tranzacționale sunt fragmentate orizontal pe regiuni; nomenclatoarele sunt replicate complet; tabelul MEDIC este fragmentat vertical.

Strategie de fragmentare

Fragmentare orizontală primară

PACIENT₁ = σ regiune='Sud' (PACIENT) → S2
PACIENT₂ = σ regiune='Vest' (PACIENT) → S3
PACIENT₃ = σ regiune='Est' (PACIENT) → S4

Fragmentare orizontală derivată

PROGRAMAREᵢ = PROGRAMARE ⋉ PACIENTᵢ
FACTURAᵢ = FACTURA ⋉ PROGRAMAREᵢ

Fragmentare verticală MEDIC

MEDIC_OP = π id, nume, prenume, cod_parafa (MEDIC) → replicat S2/S3/S4
MEDIC_HR = π id, data_angajarii, salariu (MEDIC) → exclusiv S1

Replicare completă

SECTIE, SPECIALIZARE, DIAGNOSTIC, TRATAMENT → S1, S2, S3, S4

Constrângere distribuită — modificare salariu

Înainte de UPDATE pe MEDIC_HR (S1), sistemul interoghează toate nodurile locale (S2/S3/S4) pentru facturi cu status Restant. Dacă există, mărirea este blocată.

          Raport OLAP — Performanța medicilor pe diagnostic

          Implementarea cererii complexe din analiza distribuită: „Pentru un diagnostic dat, afișați medicul, salariul, numărul de pacienți distincți și suma totală facturată, indiferent de filială.”. Cele 3 strategii rulează același business case dar cu metode diferite de execuție distribuită.
